# Agent 1: Interview Agent

> **Architectural note:** Originally designed as "Voice Conversationalist" with real-time Cypher composition. Now uses a pre-computed briefing flow — all KuzuDB queries happen before voice starts, not during.

## Role

Primary voice interface. Conducts a structured interview about the developer's codebase understanding using pre-generated questions.

## Model

**DevStral Small 2** (24B, 200 t/s) — optimized for speed (~370ms TTFT). Runs via the Express proxy.

## Capabilities

- Pre-computed interview with 5-8 questions per session
- Evaluates answers against ground truth from the briefing
- Gives conversational feedback (encouragement, corrections, hints)
- Scores the developer's overall understanding
- No real-time Cypher — all data is pre-gathered

## Briefing Pipeline

The interview flow is implemented in `src/lib/briefing.ts`:

1. **`gatherContext()`** — 6 parallel KuzuDB queries for files, functions, classes, calls, imports, contributors
2. **`generateBriefing()`** — Sends context to Mistral (JSON mode, non-streaming) to generate questions + ground truth
3. **`composeBriefingPrompt()`** — Formats into a system prompt with personality, questions, flow instructions, voice guidelines, and guardrails

## System Prompt Structure

The system prompt (generated by `composeBriefingPrompt()`) includes:

```
# Personality
Friendly codebase interview assistant, concise and encouraging

# Codebase Summary
[Auto-generated from graph data]

# Interview Questions
Question 1: [question] / Ground truth: [answer]
Question 2: [question] / Ground truth: [answer]
...

# Flow
1. Greet, announce question count
2. Ask questions one at a time
3. Evaluate answers, give feedback
4. After last question, give overall assessment

# Voice Guidelines
- Speak naturally, say file names as words
- Don't dump data, keep it conversational

# Guardrails
- Stay on topic, don't make up information
- NEVER use tool calls — just speak naturally
```

## Pre-Computed Cypher Examples

These queries run in `gatherContext()` before voice starts:

```cypher
-- Files
MATCH (f:File) RETURN f.name, f.filePath LIMIT 50

-- Functions
MATCH (fn:Function) RETURN fn.name, fn.filePath LIMIT 100

-- Call relationships
MATCH (a)-[r:CALLS]->(b) RETURN a.name, b.name LIMIT 100

-- Contributors
MATCH (p:Person)-[r:CONTRIBUTED]->(f:File) RETURN p.name, f.name LIMIT 100
```
